Karma.gs is a simple grid system written in Stylus that takes the Goldilocks approach, and includes a built-in typographic grid.

Goldilocks Approach

Karma uses the Goldilocks Approach. This is a very organic approach to thinking about responsive websites, it puts aside the battle to accommodate every possible screen size, and lets you create something that is truly flexible and will work with just about any screen size. Approach.

Typographic Scale

Just your basic grid system:

.main
        container()

      .info-section
        row()

      .about
        columns(4)

      .contact
        columns(2)

But you can use decimals!

.about
        columns(2.333)
      .contact
        columns(3.666)

Offsets and pulls:

.about
        offset(1)
        pull_right()
        columns(1.333)
      .contact
        columns(3.666)

Customize responsive behaviour:

.about
        always_expand() // Ignore **_grid_expand_min_proportion** setting
        columns(1)
      .contact
        columns(5)
      
.about
        ignore_media_queries() // Just generate a % width
        columns(1)
      .contact
        ignore_media_queries()
        columns(5)
      
.about
        expand_for 'mummy' // Expand on 'mummy' size
        columns(1)
      .contact
        ignore_media_queries()
        columns(5)

Define the scale you want to use by overriding the defaults in defaults.styl:

typographicScale = _sequence_traditional

To use a particular size:

.about
        scale_type 2
      .contact
        scale_type -1

To maintain rhythm while spacing two blocks:

.info-section
        margin-bottom line_breaks(2)

Composing via mobile-first

You should read the Goldilocks Approach before proceeding.

Karma comes with these core media query helpers:

  • @media daddy
  • @media daddyOnly
  • @media mummy
  • @media baby

Say you want to make a button have different font sizes according to the screen size. First create the button defaulting to mobile:

.button
        scale_type(-1)

And then add the rule for how it would look on large screens (e.g desktop):

.button
        scale_type(-1)

        @media daddy
          scale_type(2)

Finally if it should look different on medium screen sizes (e.g iPad), then add the mummy rule:

.button
        scale_type(-1)

        @media daddy
          scale_type(2)

        @media mummy
          scale_type(0)

Isn't it awesome how all of the relevant response rules are encapsulated in the same class? No more messing around with multiple files.

How to use

You can install this with bower:

bower install karma

Or simply download and put the files somewhere. And then import these files individually:

@import "karma/defaults"
      @import "karma/utilities"
      @import "config" // YOUR OWN config file, these can override the defaults.
      @import "karma/layouting" // Layouting Grid System
      @import "karma/typography" // Typography Grid System
